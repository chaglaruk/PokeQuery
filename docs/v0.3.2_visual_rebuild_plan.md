# PokeQuery v0.3.2 visual rebuild plan

## Current visual problems
- Onboarding is sparse: small art floats in empty navy space and does not explain the product quickly.
- Home wastes the biggest visual area on a muted header plus two CTAs, then pushes useful goal cards lower.
- Goal cards read as small utility tiles, not collectible-game cards; art is too small and abstract.
- Goal detail screens are functional but flat: risk, copy, explanation, warnings and options do not form a premium hierarchy.
- Bottom navigation is missing History.
- Settings still contains control-like rows that do not materially change behavior.

## Target differences
- Full-screen dark-blue map/route mood, with glowing pins and layered gradients.
- Large title/hero compositions and visible mascot/creature-style silhouettes.
- Two-column goal grid with large 70-140dp art and deliberate spacing.
- Detail screens prioritize risk, generated string, copy, explanation, warnings/chips, then options.
- Navigation matches Builder / Favorites / History / Knowledge / Settings.
- No official Pokémon/Niantic/Nintendo/Game Freak assets at runtime.

## Screens to rebuild
- Onboarding step 1/2/3.
- Home / Builder.
- GoalDetail for Safe Cleanup, 2x Candy, Trade Fodder, Nundo, PvP, Lucky Trade.
- Favorites, new History, Knowledge, Settings.
- App launcher icon foreground/background.

## Release-safe vs prototype-only
- Release-safe: Compose/vector map lines, shield, magnifier, abstract mascot silhouettes, generic cards, local `knowledgebase.json`.
- Prototype-only: any user-supplied official or lookalike character art. None will be committed in this pass.

## Acceptance criteria
- Build/test and runtime asset guard pass.
- App icon is visibly stronger and adaptive icon resources resolve.
- Onboarding and home are closer to `docs/design/reference_target_pokequery.png`.
- No dead blank block under the Home CTAs.
- Goal cards use large meaningful art, not tiny icon clutter.
- Goal detail copy flow and live options remain intact.
- Back navigation uses fade-only transitions, not scale/shrink.
- History tab is functional or explicitly deferred.
- Deterministic screenshots use an explicit allowlist and fail on missing required images.
