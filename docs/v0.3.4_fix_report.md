## 3. String Localization Behavior
The `SearchTermMapper.kt` logic correctly translates generated valid syntax depending on the `Game Language` chosen by the user. Integers attached to logical components (`count`, `distance`) are cleanly uncoupled, leaving the integers completely intact.
- `legendary` -> `efsanevi`
- `shiny` -> `parlak`
- `traded` -> `takaslanan`
- `count2-` -> `toplam2-`
- `0attack&0defense&0hp` -> `0saldırı&0savunma&0can`
The mapping has no uncertain tokens left to address. See `search_language_mapping.md` for the entire table.

## 4. Knowledge Base Refinements

All major search terms (`&`, `,`, `!`, `shiny`, `legendary`, `lucky`, `shadow`, `purified`, `count`, `cp`, `hp`, `age`, `distance`, `0*`) now contain simple, plain English summaries along with explicit examples, cautions, and language mapping caveats.
